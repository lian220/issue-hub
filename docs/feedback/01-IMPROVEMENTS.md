# IssueHub 프로젝트 보완 및 개선 제안서

> 문서 버전: 1.0
> 작성일: 2026-03-21
> 작성자: IssueHub AI 어시스턴트
> 목적: 현재까지 작성된 PRD, 아키텍처, 로드맵을 기반으로 한 식별된 누락 사항 및 개선 권고사항 정리

---

## 1. UX/UI 및 프론트엔드 설계의 부재 (가장 시급함)

**현황**: 
백엔드 아키텍처(헥사고날 구조, 이벤트 처리)와 연동 스펙은 매우 구체적이나, 정작 최종 사용자가 겪게 될 **화면(UI) 구조와 컴포넌트 설계 가이드가 전무**합니다. `Next.js`와 `Tailwind CSS`를 사용한다는 기술 스펙만 있을 뿐, 아래 내용이 비어 있습니다.

**보완 권고**:
*   **디자인 시스템 도입**: shadcn/ui나 Base UI 등을 베이스로 하되, IssueHub만의 컬러 팔레트, 타이포그래피, 버튼 컴포넌트 등 디자인 시스템(Figma)을 먼저 정의해야 합니다.
*   **와이어프레임 설계**: 칸반 보드, 통합 대시보드, 티켓 상세 페이지 등 핵심 페이지에 대한 레이아웃 설계가 백엔드 개발 전에 선행되어야 API 명세(Response 형태)를 정확하게 확정할 수 있습니다.

## 2. 권한 시나리오의 구체성 부족 (RBAC 예외 처리)

**현황**: 
Admin, Manager, Member, Viewer 4가지 역할이 정의되어 있으나, **팀 간 교차 권한(Cross-team Permissions)**에 대한 시나리오가 없습니다.

**보완 권고**:
*   A팀의 Manager가 B팀의 티켓 상태를 바꿀 수 있는지?
*   특정 정책 문서를 특정 부서(예: 인사팀)만 열람하게 하는 기밀(Confidential) 권한이 있는지?
*   이러한 세밀한 권한 제어(ABAC - Attribute Based Access Control) 요건이 추후 발생할 가능성이 높으므로, 초기 DB 설계(권한 매트릭스)에 확장성을 고려해야 합니다.

## 3. PII (개인정보) 및 민감 데이터 처리의 구체화

**현황**: 
보안 리뷰 문서 보관 및 마스킹(Masking) 요건이 언급되어 있으나, **어디서 어떻게 마스킹할지** 흐름이 불분명합니다.

**보완 권고**:
*   Jira/GitHub에서 넘어오는 웹훅 데이터에 PII가 있을 경우 DB에 저장되기 전(in-flight)에 마스킹할 것인지, 아니면 원본은 암호화하여 저장하고 화면에 프론트엔드가 렌더링할 때 마스킹할 것인지 아키텍처 결정을 내려야 합니다. 
*   추천: `core-issue` 도메인 내부에 `PiiSanitizer` 포트를 두어 DB 저장 전 마스킹을 원칙으로 하는 것을 제안합니다.

## 4. 알림 피로도 (Notification Fatigue) 방지 대책

**현황**: 
이슈 변경, 슬랙 알림, 에스컬레이션 등 알림 기능이 굉장히 많습니다. 사용자가 알림의 홍수에 빠질 위험이 큽니다.

**보완 권고**:
*   **알림 그룹화(Batching / Digest)**: 매 변경 건마다 알림을 보내는 것 외에도 "하루에 2번 요약본만 받기(Digest)" 옵션의 세부 로직이 필요합니다.
*   **스마트 무음(Smart Mute)**: 사용자가 휴가 중이거나, 주말/심야 시간대에는 알림을 어떻게 처리할지(지연 발송 또는 대리자에게 라우팅) 규칙 엔진에 반영해야 합니다.

## 5. RAG AI 도입에 따른 토큰 비용 설계 (Phase 3)

**현황**: 
pgvector를 이용한 문서 하이브리드 검색 및 Q&A 봇이 계획되어 있으나, OpenAI 등 외부 LLM을 사용할 경우의 **비용 통제(Cost Control) 계획이 없습니다**.

**보완 권고**:
*   비용 폭발을 막기 위해 1) 로컬 캐싱(Redis 질의응답 캐시)을 두어 동일 질문에는 LLM을 호출하지 않게 하거나, 2) 부서/사용자별 월간 토큰 제한(Quota)을 두는 기능을 로드맵 Phase 3에 포함해야 합니다.

현재 프로젝트는 엔지니어링 완성도는 훌륭하나 **"제품(Product)이 고객에게 주는 경험적 디테일"**이 약간 부족합니다. 당장 코드 작성에 들어가기 전, UX/UI 전문가를 영입해 프론트엔드 비주얼라이제이션(Visualization)을 확정하는 것이 성공의 필수 조건입니다.
