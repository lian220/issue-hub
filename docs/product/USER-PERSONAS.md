# IssueHub - 사용자 페르소나 및 권한 정의

> 문서 버전: 2.0
> 최종 수정일: 2026-03-28
> 작성자: IssueHub 개발팀
> 상태: 검토 중

---

## 1. 사용자 규모

- **조직 규모**: 10~50명
- **대상**: 전사 조직 (개발팀 + 기획/운영/CS 등 비기술 조직)

---

## 2. 사용자 페르소나


| 페르소나          | 역할                   | 주요 니즈                                   |
| ------------- | -------------------- | --------------------------------------- |
| **개발 리드**     | 기술 이슈 관리, 코드 리뷰 연동   | GitHub PR/Issue와 Jira 양방향 동기화, SLA 모니터링 |
| **PM/기획자**    | 전체 이슈 현황 파악, 우선순위 조정 | 통합 대시보드, 트렌드 분석, 팀 워크로드 시각화             |
| **운영/CS**     | 고객 이슈 접수 및 추적        | Slack/Teams/Discord 알림, 자동 분류, 에스컬레이션    |
| **컴플라이언스 담당** | 정책 관리 및 준수 추적        | 정책 버전 관리, 읽음 확인, 위반 감지                  |
| **관리자**       | 시스템 설정 및 권한 관리       | RBAC, 커넥터 설정, 자동화 규칙 관리                 |
| **일반 개발자** | 이슈 할당받아 코드 작성, AI 분석 활용 | AI 코드 분석 결과 확인, 이슈 상태 업데이트, 관련 코드 컨텍스트 습득, 자동 개발 요청 |


### 페르소나-역할 매핑

| 페르소나 | 역할 |
| --- | --- |
| 관리자 | Admin |
| 개발 리드 | Manager |
| PM/기획자 | Manager |
| 컴플라이언스 담당 | Manager |
| 운영/CS | Member |
| 일반 개발자 | Member |
