## 6. 비기능 요구사항

### 6.1 성능


| 항목         | 요구사항                     |
| ---------- | ------------------------ |
| API 응답 시간  | 95th percentile 500ms 이내 |
| 웹훅 처리 지연   | 5초 이내                    |
| SSE 이벤트 전달 | 2초 이내                    |
| 동시 사용자     | 50명 동시 접속 지원             |
| 이슈 검색      | 10만건 기준 1초 이내 (OpenSearch) |


### 6.2 실시간 업데이트

- Server-Sent Events (SSE) 기반 실시간 푸시
- Redis Pub/Sub를 통한 이벤트 분산
- 이슈 상태 변경, 댓글 추가, 동기화 완료 등 실시간 반영

### 6.3 배포 환경

- 온프레미스 배포 지원 (Docker Compose)
- 클라우드 배포 지원 (Kubernetes)
- 환경별 설정 분리 (dev, staging, production)

### 6.4 보안


| 항목     | 요구사항                          |
| ------ | ----------------------------- |
| 인증     | Keycloak 기반 OAuth2/OIDC       |
| 토큰 저장  | AES-256 암호화 (외부 플랫폼 OAuth 토큰) |
| 웹훅 검증  | HMAC-SHA256 서명 검증             |
| API 인증 | JWT Bearer Token              |
| 데이터 전송 | TLS 1.2 이상                    |
| 감사 로그  | 모든 변경 작업 기록                   |


### 6.5 가용성

- 목표 가용성: 99.5% (계획된 점검 제외)
- 장애 복구 시간(RTO): 4시간 이내
- 데이터 복구 시점(RPO): 1시간 이내

## 7. 보안/컴플라이언스 요구사항

> 전문가 패널 리뷰를 반영하여 추가된 섹션


| ID      | 요구사항                 | 우선순위 | 설명                                         |
| ------- | -------------------- | ---- | ------------------------------------------ |
| SEC-001 | 감사 로그 보관 정책          | P1   | 기본 1년 보관, 규제 산업 5년 보관, 아카이빙 및 삭제 절차 정의     |
| SEC-002 | 개인정보 보호              | P1   | 이슈 내 고객 정보 마스킹, 민감정보 태깅, PIPA(개인정보보호법) 준수  |
| SEC-003 | 토큰/시크릿 로테이션 정책       | P1   | ENCRYPTION_KEY 등 시크릿의 주기적 변경 절차 및 무중단 로테이션 |
| SEC-004 | 접근 권한 변경 감사          | P1   | 역할 변경, 권한 부여/회수 이력의 명시적 감사 로그 기록           |
| SEC-005 | API Rate Limiting    | P2   | 사용자별/IP별 분당 요청 수 정의 및 초과 시 차단 정책           |
| SEC-006 | 데이터 at-rest 암호화      | P2   | PostgreSQL TDE 또는 볼륨 수준 암호화 적용             |
| SEC-007 | 세션 관리                | P1   | JWT 만료 시간 설정, 리프레시 토큰 정책, 동시 로그인 제한        |
| SEC-008 | IP 화이트리스트/네트워크 접근 제한 | P3   | 관리자 설정을 통한 허용 IP 대역 관리                     |

## 8. 성공 지표 (KPI)


| 지표           | 목표                         | 측정 방법                  |
| ------------ | -------------------------- | ---------------------- |
| 이슈 통합 커버리지   | 90% 이상의 이슈가 IssueHub에서 추적됨 | 플랫폼별 이슈 수 대비 동기화된 이슈 수 |
| 평균 이슈 라우팅 시간 | 도입 전 대비 70% 감소             | 이슈 생성 → 담당자 배정까지 소요 시간 |
| SLA 준수율      | 85% 이상                     | SLA 정책 대비 기한 내 해결 비율   |
| 수동 작업 감소율    | 50% 이상 감소                  | 자동화 적용 전후 수동 작업 건수 비교  |
| 정책 읽음률       | 80% 이상                     | 게시된 정책 대비 대상자 읽음 확인 비율 |
| 사용자 만족도      | 4.0/5.0 이상                 | 분기별 사용자 설문             |
| 시스템 가용성      | 99.5% 이상                   | 모니터링 도구 기준 업타임         |


### 8.1 성공 측정 타이밍


| 시점              | 측정 항목                           |
| --------------- | ------------------------------- |
| Phase 1 배포 2주 후 | 개발팀 채택률 측정 (활성 사용자 수, 일일 접속 빈도) |
| Phase 2 배포 4주 후 | 자동화 규칙 활성화율, 수동 작업 감소율 측정       |
| Phase 3 배포 8주 후 | 전체 KPI 대비 달성도 평가 (위 지표 전체 측정)   |
| 6개월 후           | 종합 ROI 분석 및 지속/중단 의사결정          |

## 9. 제약사항 및 가정

### 9.1 제약사항

- Jira Cloud 및 GitHub Cloud 환경을 우선 지원 (Server/Data Center는 Phase 4 이후 검토)
- RAG 기반 AI 기능은 Google Gemini API (최신 안정 버전) 사용을 전제
- 단일 테넌트 구조로 시작 (멀티 테넌트는 향후 검토)

### 9.2 가정

- 대상 조직이 Jira, GitHub 중 하나 이상을 사용 중
- Keycloak 또는 호환 OIDC Provider 운영 가능 (Phase 3 이후)
- PostgreSQL 16 이상 사용 가능 (Aiven 무료 플랜으로 제공)
- OpenSearch 로컬 실행을 위한 Docker 환경 구성 가능
- 대상 사용자 규모: 50명 이상으로 확장 가능성 고려
