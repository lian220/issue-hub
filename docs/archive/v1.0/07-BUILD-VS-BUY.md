### Buy 옵션들

#### Option A: Jira Service Management + Jira Automation + Confluence

**구성**:
- Jira Service Management (Cloud Premium): 이슈 관리, SLA 추적, 워크플로우
- Jira Automation: 자동화 규칙, 조건부 트리거
- Confluence (Standard): 정책 문서 관리, 지식베이스

| 규모 | 월간 비용 | 3년 TCO | PRD 커버리지 |
|------|-----------|---------|-------------|
| 10명 | 약 52만원 | 약 1,872만원 | 65-70% |
| 30명 | 약 144만원 | 약 5,184만원 | 65-70% |
| 50명 | 약 242만원 | 약 8,712만원 | 65-70% |

**미커버 영역**:
- 정책 승인 워크플로우 (다단계 결재 로직)
- 읽음 확인 (정책 배포 후 확인 추적)
- 컴플라이언스 추적 (규제 매핑, 증적 관리)
- RAG 기반 Q&A 봇 (AI 기반 자연어 질의)
- PII 마스킹 (개인정보 자동 탐지/처리)
- 온프레미스 배포 (Data Center 라이선스 별도, 비용 2-3배)

**장점**: 검증된 플랫폼, 방대한 에코시스템, 빠른 도입
**단점**: 라이선스 비용 증가 추세 (연 10-15% 인상), 커스터마이징 한계, 데이터 이관 어려움

| 50명 | 약 187만원 | 약 6,732만원 | 55-60% |

**미커버 영역**: Option A의 미커버 영역 + SLA 관리 한계, 복잡한 워크플로우 제약, 감사 로그 부족

**장점**: 모던 UX (특히 Linear), 개발팀 친화적, 빠른 셋업
**단점**: 엔터프라이즈 기능 부족, Zapier 비용 급증 가능 (작업 수 기반), 대규모 조직 확장성 제한

**장점**: 최저 비용, 비개발자 친화적 UI, 빠른 도입
**단점**: 이슈 관리 전문성 부족, 복잡한 워크플로우 구현 한계, 개발팀 적합성 낮음

### Hybrid 옵션

**전략**: 이슈 통합/자동화는 SaaS 활용, 정책 관리/컴플라이언스만 자체 개발

**구성**:
- **Buy 영역**: Jira Cloud (Premium) + Zapier/Make (이슈 통합, 자동화, 알림)
- **Build 영역**: 정책 관리 모듈 + 컴플라이언스 추적 + PII 마스킹 (12-16주, 2.5명)

#### Hybrid 비용 상세 (30명 기준)

| 항목 | 산출 근거 | 비용 |
|------|-----------|------|
| **Buy 파트 (연간)** | | |
| Jira Cloud Premium 30명 | 월 약 90만원 | 약 1,080만원 |
| Zapier Professional | 월 약 15만원 | 약 180만원 |
| **Buy 소계 (3년)** | | **약 3,780만원** |
| **Build 파트** | | |
| 초기 개발 (16주, 2.5명) | 평균 연봉 7,000만원 x 2.5 x 4/12 | 약 5,830만원 |
| 유지보수 (연간, 0.5명) | 연봉 7,000만원 x 0.5 | 약 3,500만원 |
| 인프라 (연간) | 서버 1대 + DB | 약 480만원 |
| **Build 소계 (3년)** | | **약 1억 7,790만원** |
| **Hybrid 3년 TCO** | | **약 2.8억원** |
| **PRD 커버리지** | | **90-95%** |

**Hybrid의 장점**:
- 핵심 차별화 기능만 자체 개발하여 개발 리스크 최소화
- SaaS 부분은 즉시 활용 가능 (Time-to-Value 단축)
- 유지보수 인력 부담 대폭 감소 (4-5명 → 0.5명)
- SaaS 기능 업데이트를 무료로 활용

**Hybrid의 단점**:
- 두 시스템 간 데이터 동기화 복잡성
- SaaS 부분 벤더 종속 존재
- 사용자 경험 분리 가능성 (두 시스템 전환)

## 3. 시나리오별 권고

### 시나리오 A: 규제 산업 (금융, 의료, 공공)

**권고: Build 또는 Hybrid**

**규제 환경 특성**:
- 데이터 주권 요구: 국내 데이터센터 필수 (개인정보보호법, 금융위원회 규정)
- 온프레미스 필수: 클라우드 사용 시에도 전용 인프라 요구 가능
- 감사 로그 보관: 금융권 5년, 의료 10년, 공공 5-10년
- PII 통제: 개인정보 처리 전 과정에 대한 완전한 통제 필요
- 제3자 데이터 이전 제한: SaaS 벤더에 대한 데이터 처리 위탁 계약 필요

**세부 권고**:
- **30명 이하**: Hybrid (온프레미스 Jira Data Center + 정책/컴플라이언스 자체개발)
  - Jira DC 라이선스: 연 약 2,000만원 (500 User 기준, 소규모도 동일)
  - 정책 모듈 자체개발: 12-16주
  - 3년 TCO: 약 3.5~4억원
- **50명 이상**: Full Build
  - Jira DC 비용이 자체 개발 대비 이점 감소
  - 완전한 커스터마이징으로 규제 대응 유연성 확보
  - 3년 TCO: 약 6~7억원 (규모의 경제 효과)

### 시나리오 B: IT/테크 기업

**권고: Buy 우선, 필요 시 Hybrid**

**환경 특성**:
- 핵심 제품 개발에 엔지니어링 리소스 집중 필요
- 빠른 도입과 변경이 중요 (Agile/DevOps 문화)
- SaaS 사용에 대한 조직 친화도 높음
- 보안/컴플라이언스 요구는 중간 수준

**세부 권고**:
- **즉시**: Buy (Option A 또는 B) 도입으로 빠른 가치 실현
- **6-12개월 후**: 정책/컴플라이언스 니즈 발생 시 Hybrid 전환 검토
- **핵심 원칙**: 개발 리소스를 이슈 관리 도구가 아닌 핵심 제품에 투자
- **ROI 관점**: 엔지니어 1명의 핵심 제품 기여 가치 > 이슈 관리 도구 개발 가치

### 시나리오 C: 일반 사업회사

**권고: Buy (Option A)**

**환경 특성**:
- IT는 지원 기능이지 수익 기능이 아님
- 내부 개발 역량 제한적
- 표준화된 프로세스 선호
- 비용 효율성 최우선

**세부 권고**:
- SaaS 비용 대비 자체 개발 인건비 10배 이상 차이
- Jira + Confluence 조합으로 대부분의 이슈 관리 니즈 충족
- 추가 자동화 필요 시 Zapier/Make로 확장
- 자체 개발은 권장하지 않음 (유지보수 인력 확보 어려움)

## 5. 최종 권고

### 대부분의 조직에 Hybrid를 권고

**핵심 논리**: 이슈 통합/자동화는 이미 성숙한 SaaS 시장이 존재하므로 재발명할 필요 없음. 그러나 정책 관리/컴플라이언스는 조직마다 고유한 요구사항이 있어 커스터마이징이 필수적.

**Hybrid가 최적인 이유**:
1. **비용 효율**: Full Build 대비 60% 이상 비용 절감
2. **리스크 분산**: 개발 범위 축소로 일정/품질 리스크 감소
3. **빠른 가치 실현**: SaaS 부분 즉시 사용 가능
4. **유연성 확보**: 향후 필요 시 Build 영역 확장 가능

**예외 상황**:
- **규제 산업 (온프레미스 필수)**: Build가 정당화됨
